Define class `AdaptiveOrientationAB(Node)`:
- `__init__`: super-init with name `'adaptive_orientation_ab'`. Declare two parameters (`adaptive` default `True`, `timeout_sec` default `3.0`); read both into `self.adaptive_label` and `self.timeout_sec`. Create publisher `self.goal_pub = self.create_publisher(PoseStamped, '/goal_pose', 10)`. Create subscriber `self.traj_sub = self.create_subscription(JointTrajectory, '/joint_trajectory_targets', self._traj_cb, 10)`. Initialize state: `self.results = []  # list of (label, x, y, z, passed)`, `self._traj_received_for_current = False`, `self._current_index = 0`. Log a startup INFO line stating the configured `adaptive` label and `timeout_sec`. Use `self.create_timer(0.5, self._tick)` to run the driver loop (NOT `rclpy.spin` directly — the timer drives the per-target sequence so we yield to ROS message processing between targets). Use a small initial settle delay (e.g., 1.0 s) before publishing the first goal so that the publisher's connection is established to the planner subscriber.

- `_traj_cb(self, msg)`: set `self._traj_received_for_current = True` (presence-only — we do not validate trajectory contents in this harness, since the planner already validates via its existing pre-publish auto-fix block).

- `_publish_goal(self, label, x, y, z)`: build a `PoseStamped`, set `header.frame_id = 'torso_link'`, `header.stamp = self.get_clock().now().to_msg()`, set position xyz, set orientation to `BASELINE_QUAT`. Publish once. Reset `self._traj_received_for_current = False` BEFORE publishing.

- `_tick(self)`: advance the driver state machine. Maintain a phase enum (e.g., `_phase = 'settle'|'publishing'|'waiting'|'done'`). On `settle`, just count down 1.0 s, then move to `publishing`. On `publishing`, call `_publish_goal` for `TARGETS[self._current_index]`, record the publish wall time, move to `waiting`. On `waiting`, check whether `self._traj_received_for_current` flipped; if yes, append `(label, x, y, z, True)` to results and advance index; if no AND elapsed > `self.timeout_sec`, append `(label, x, y, z, False)` and advance index. When `self._current_index == len(TARGETS)`, log the summary (per-target rows + `PASS_COUNT/8 — adaptive=<label>`), set `_phase = 'done'`, and call `self._exit_status = 0 if all_passed else 1`. Use a flag `self._finished = True` to break the spin in main.

- `main(args=None)`: standard `rclpy.init(args=args)`; construct the node; drive the event loop using `rclpy.spin_once(node, timeout_sec=0.1)` inside a `while rclpy.ok() and not node._finished:` loop (do NOT use `rclpy.spin(node)` — the harness must terminate after the 8-target sequence completes). Capture `node._exit_status` before teardown. Wrap the loop in `try/except KeyboardInterrupt` that sets exit_status to 130 (standard SIGINT exit code). In `finally`, call `node.destroy_node(); rclpy.try_shutdown()`. Last line: `sys.exit(exit_status)`.

The script body MUST be self-contained — no imports from elsewhere in the package. The harness MUST NOT subscribe to or import anything related to the joint trajectory executor (`unitree_hg`, `joint_trajectory_executor`, etc.) — D-16 forbids running the executor for Phase 8 verification.

Make the file executable on disk (`chmod +x`). The CMake `install(PROGRAMS …)` step (Task 2) preserves the executable bit during install.

## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| Operator → harness CLI args / ROS params | `adaptive` and `timeout_sec` parameters; trusted same as the launching shell. |
| Harness → ROS DDS `/goal_pose` publisher | The harness becomes a goal-publisher on the same domain as the planner. Same boundary as `keyboard_trigger_node.py`. |
| Planner → harness `/joint_trajectory_targets` subscriber | Planner trajectory output. The harness only checks for *presence* of a message; it does not parse, validate, or act on trajectory contents. |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|---|---|---|---|---|
| T-08-05 | Spoofing | A rogue process on the same ROS domain could publish bogus `/joint_trajectory_targets` to the harness, causing false-positive PASS counts. | accept | The harness is run interactively by an operator on the same machine as the planner. Any ROS domain mis-configuration that allows a rogue publisher would already break Phase 6/7 verification harnesses identically. No new exposure. |
| T-08-06 | Denial of Service | The harness loops over a fixed 8-target list with a 3 s timeout; total runtime is bounded at ≤ 30 s. | accept | Bounded by design — there is no unbounded retry, no recursion, no infinite loop. The `_finished` flag and the `sys.exit(exit_status)` guarantee termination. |
| T-08-07 | Tampering | Operator could be tempted to silently raise `timeout_sec` to mask planner regressions. | accept | The harness logs the configured `timeout_sec` at startup, and the recorded UAT entry in `08-VERIFICATION.md` is expected to capture the parameter value. Procedural mitigation, not technical. |
| T-08-08 | Elevation of Privilege | The harness must NOT bypass the executor decoupling (D-16). A bug that imported `unitree_hg` or talked to the executor would let the harness move the robot. | mitigate | The acceptance criteria includes a negative grep for `unitree_hg\|joint_trajectory_executor\|JointTrajectoryExecutor\|Dex3Controller` — the harness CANNOT contain those symbols. Static check enforced at the source level. The loose substring `dex3` is intentionally NOT matched because the package directory name `unitree_g1_dex3_stack` legitimately contains it. |
| T-08-SC | Tampering | npm/pip/cargo supply chain. | N/A | Plan 02 installs zero new packages. The harness imports rclpy and standard ROS message packages already declared in `package.xml`. |
</threat_model>
