### Task 1: Add tcp_offset_x parameter and chain override to planner C++ code
<read_first>
- src/unitree_g1_dex3_stack-main/src/ik_fcl_ompl_planner.cpp
- .planning/phases/06-yolo-tcp-offset/06-RESEARCH.md (Section 3: KDL Chain Runtime Override)
- .planning/phases/06-yolo-tcp-offset/06-PATTERNS.md (ik_fcl_ompl_planner.cpp section)
</read_first>

<action>
In src/unitree_g1_dex3_stack-main/src/ik_fcl_ompl_planner.cpp:

1. Change the default value in the declare_parameter call for "right_tip" from "right_wrist_yaw_link" to "right_tcp_link" (line ~65).

2. After the successful getChain() call (after the block ending with `return;` on failure, around line ~131), insert the tcp_offset_x parameter declaration and chain override logic:
   - Declare parameter "tcp_offset_x" with default 0.175 (double)
   - Get the parameter value into a local double variable tcp_offset_x
   - Get the number of segments in kdl_chain_right
   - Check if the last segment has Joint::None type (fixed joint)
   - If yes: rebuild the chain by copying all segments except the last, then append a new KDL::Segment with the same name as the last segment, Joint::None, and KDL::Frame(KDL::Vector(tcp_offset_x, 0.0, 0.0))
   - Assign the rebuilt chain back to kdl_chain_right
   - Log an INFO message: "TCP offset overridden to %.4f m" with the tcp_offset_x value

This block MUST appear before the joint limits iteration loop that uses kdl_chain_right.getNrOfSegments() and before TRAC-IK construction.

3. No other changes to the file. The /goal_pose subscription, FK solver, OMPL planning logic all remain unchanged — they automatically use the modified chain.
</action>
